## Terraform deployment

:::info Development workspace
- Deploy the **dev** workspace using Terraform.
- Take in consideration the tenant and resource group where it's supposed to be deployed.
  - **Example:** dev subscription, the ***dev workspace*** is going to be deployed inside the ***databricks-resources*** resource group.
- Use the following Terraform template: [01-azure-workspace-vnet-injection](https://github.com/ivancalvo-dbxs/terraform-databricks-workspaces-2025/tree/main/azure/01-azure-workspace-vnet-injection)
:::

:::info Staging and Prod workspaces
* Repeat the process for the **staging** and **production** workspaces.
- Take in consideration the tenant and resource group where it's supposed to be deployed.
  - **Example:** staging subscription, the ***staging workspace*** is going to be deployed inside the ***databricks-resources*** resource group.
* Use the following Terraform template: [01-azure-workspace-vnet-injection](https://github.com/ivancalvo-dbxs/terraform-databricks-workspaces-2025/tree/main/azure/01-azure-workspace-vnet-injection)
:::
